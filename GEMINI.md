# ひらめき365：開発・運用ルール

このプロジェクトを安全かつ一貫性を保って開発するための基礎ルールです。

## 1. バージョン管理の徹底
- **一元管理**: アプリケーションのバージョンは `package.json` の `version` フィールドで一括管理しています。
- **自動反映**: トップページ（index.astro）およびクイズ詳細ページ（[slug].astro）のフッターには、`package.json` の値が自動的に反映されます。
- **更新手順**: 
    1. `package.json` の `version` を更新する。
    2. Gitでタグを打つ（例：`git tag v0.2.1`）ことで、リリースの履歴を管理する。
- **一貫性の保持**: 開発環境と本番環境での混乱を防ぐため、修正を加えた際は必ずバージョンをインクリメント（加算）してください。

## 2. 実装と検証のルール
- **全画面検証**: UIや判定ロジックに変更を加えた後は、必ず「カレンダー画面」と「クイズ画面」の両方で動作を確認してください。特にスマホ表示（レスポンシブ）の崩れがないか、双方で検証する必要があります。
- **localStorageの整合性**: データの保存形式（hirameki365-resultsなど）を変更した場合は、古いデータ形式との互換性を維持するか、移行ロジックを必ず含めてください。

## 3. シニア向けUI/UXの堅持
- **文字サイズ**: 常に大きなフォント（標準で1.2rem以上）を基本とし、視認性の高い配色（WCAG AA準拠）を維持してください。
- **操作の簡略化**: 二重クリックの防止（今回のスコア集計修正など）や、オートフォーカスなど、ユーザーのユーザーの操作負担を最小限にする工夫を継続してください。

## 4. クイズの質と監査ルール（重要）
コンテンツの品質を維持するため、以下の監査ルールを遵守してください。
- **選択肢の関連性**: Q1（昭和・平成回想）などの選択肢は、必ず問題のカテゴリと一致させてください。
    - 例：「何市ですか？」という問いには、選択肢すべてを「市」の名前にする。「最高法規は何ですか？」には、他の法規（民法、刑法など）を並べる。
- **表記ゆれの許容**: Q5などの記述式回答では、複数の正解（例：「ごはん|食事」）を設定し、ユーザーの入力が柔軟に判定されるようにしてください。
- **言葉パズルの難易度**: 文字の並び替え問題は、原則として**5文字以上**の言葉を使用してください（3文字などは簡単すぎるため避ける）。
- **解説の提供**: 歴史的な事実や豆知識的な問いについては、正解とともに簡潔な「解説」を表示できるようにしてください。
